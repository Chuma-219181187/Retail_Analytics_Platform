PART 5: CREATE VIRTUAL ENVIRONMENT (IMPORTANT!)
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Open VS Code Terminal

STEP 2: Create Virtual Environment

  Windows:

✓ You should see (venv) at the start of the terminal line

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

PROBLEM: "(venv) not appearing in terminal"
SOLUTION: 
  - Windows: Make sure you ran: venv\Scripts\activate
  - Mac/Linux: Make sure you ran: source venv/bin/activate
  - Try closing and reopening terminal

PROBLEM: "ModuleNotFoundError: No module named 'pandas'"
SOLUTION:
  - Check (venv) is activated (should see it in terminal)
  - Run: pip install -r requirements.txt again
  - Wait for "Successfully installed..." message

PROBLEM: "FileNotFoundError: data/sales.csv"
SOLUTION:
  - Make sure you ran: python data_generation.py FIRST
  - Check ./data/ folder exists in file explorer
  - Check terminal for "✓ DATA GENERATION COMPLETE!"

PROBLEM: "Dashboard shows blank"
SOLUTION:
  - Hard refresh browser: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
  - Try different browser
  - Check that analytics_engine.py has been run

PROBLEM: "Script runs very slowly"
SOLUTION:
  - This is normal for first run (data generation is slow)
  - Subsequent runs are faster
  - Close other applications to free up RAM
